## 今日要点

1. **开源生态仍在持续迭代。**截至核验时，OpenAI Agents SDK 与 A2A 的 GitHub 仓库都显示在 7 月 19 日有更新；Google ADK Python 仓库则在 7 月 18 日更新。活跃度本身不等于功能发布，但说明这些工程入口仍在快速演进。
2. **产品形态向“持续代理”延伸。**Google 5 月公布的 Search agents 设想强调持续监测信息、生成综合更新，并在用户授权的场景下协助采取行动。
3. **评测应覆盖真实工作流。**TheAgentCompany 将浏览网页、写代码、运行程序和与同事沟通纳入同一模拟工作环境；论文报告的最强基线仍只能自主完成 30% 的任务，提醒团队谨慎设定自动化边界。

## 论文 / 研究

### TheAgentCompany：用“数字员工”任务检验 Agent

- **论文：**[TheAgentCompany: Benchmarking LLM Agents on Consequential Real World Tasks](https://arxiv.org/abs/2412.14161)（arXiv v3，2025-09-10）
- **研究内容：**论文构建模拟小型软件公司的自包含环境，让 Agent 通过网页浏览、编码、运行程序与沟通协作来完成任务。
- **核心结果：**作者报告最强的基线 Agent 可自主完成 **30%** 的任务；更长程、复杂任务仍是明显瓶颈。
- **为什么值得关注：**这类评测比单轮问答或单一代码修复更接近实际部署：它迫使系统面对状态、工具、协作与后果，并给产品团队一个更谨慎的自动化上限参考。

## 业界动态

### Google Search 的信息型 Agent 路线图

- **官方公告：**[A new era for AI Search](https://blog.google/products-and-platforms/products/search/search-io-2026/)（Google，2026-05-19）
- **已公布内容：**Google 表示将从 information agents 开始：Agent 可在后台持续检索网页、新闻、社交内容与部分实时数据，在发生变化时提供综合更新；公告称其计划于今年夏季先向 Google AI Pro 与 Ultra 订阅用户推出。
- **为什么值得关注：**这是从“用户发问、系统作答”转向“用户定义目标、系统持续观察”的产品交互变化。对开发者而言，权限、通知频率、来源可追溯性与停止条件会成为体验与安全的共同约束。

## 趋势 / 编辑简评

今天的信号不是某个单点模型发布，而是 Agent 走向生产化时的三个基础设施层：**工作流框架、互操作协议、长期任务评测**。它们共同指向一个现实：把模型接入工具只是开始；真正难的是让多个系统在清晰权限、可观测状态和可验证结果下长期运行。

对于团队的实操建议是：优先为高价值流程补齐评测与人工接管路径，再考虑扩大自治范围；跨团队集成时，先定义任务、能力与结果的接口契约，而不要把所有协作逻辑锁死在单一 Agent 框架中。
