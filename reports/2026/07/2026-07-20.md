## 今日要点

1. **Agent 的控制结构正在外显。**pi-for-each 将目录或文件行映射为严格串行的提示词迭代；WhichRules 则把 Agent 运行前的规则来源和冲突列出来。
2. **“可解释”从回答走到执行条件。**Semantic Agent 强调语义模型、只读查询与返回行可追溯；WhichRules 给出规则文件和行号证据。二者都在缩小“模型看到了什么、为何这样执行”的黑箱。
3. **研究与产品公告不凑数。**核验窗口内未找到可由公开一手页面确认、同时符合首次公开口径的 Agent 新论文或大型产品公告，相关栏目如实留空。

## 开源项目

### 1. pi-for-each：把批量提示词循环交给显式命令

![pi-for-each 的目录迭代演示](https://raw.githubusercontent.com/jejay/pi-for-each/main/demo-imgs/pi-for-demo-1.png)

*图：pi-for-each 官方 README 中的目录迭代演示截图。原始来源：[jejay/pi-for-each](https://github.com/jejay/pi-for-each)。*

- **首次公开：**GitHub 仓库创建于 **2026-07-20 07:20 UTC**。
- **项目：**[jejay/pi-for-each](https://github.com/jejay/pi-for-each)（MIT License）
- **项目自述：**为 [pi](https://github.com/earendil-works/pi) 增加 `/for` 命令；`$each@<路径>` 可逐个展开目录子项或文件行。循环严格串行，后续轮次会分叉会话，从而不把其他轮次的上下文放入当前提示。
- **为什么值得关注：**很多“多子任务 Agent”实现把循环隐含在长上下文或复杂编排中。这个项目选择一个小而明确的控制原语：由用户定义遍历集合、由工具管理逐轮会话隔离。它并不证明能避免所有上下文问题，但为可预测的批处理提示提供了可试验的实现。

### 2. Semantic Agent：用语义层约束自然语言数据问答

- **首次公开：**GitHub 仓库创建于 **2026-07-20 04:33 UTC**。
- **项目：**[qiboyu301-crypto/semantic-agent](https://github.com/qiboyu301-crypto/semantic-agent)（Apache-2.0）
- **项目自述：**项目基于 AgentScope 与 Wren MDL。使用者先连接数据源、审阅并激活语义模型；运行时 Agent 检索相关 schema 上下文，生成和验证只读语义 SQL，再根据查询返回行作答或绘图。README 列出 PostgreSQL、MySQL、BigQuery 等 11 类数据源。
- **为什么值得关注：**企业数据 Agent 的难点不是只把 SQL 生成得通顺，而是业务定义、权限边界与答案证据能否被复查。将“语义层优先、只读、结果有据”写进主流程，是比单纯 Text-to-SQL 更可审计的产品方向；实际权限模型、跨库正确性及部署安全仍需独立验证。

### 3. WhichRules：先弄清 Agent 会收到哪些规则

- **首次公开：**GitHub 仓库创建于 **2026-07-20 07:03 UTC**。
- **项目：**[z35068037/whichrules](https://github.com/z35068037/whichrules)（MIT License）
- **项目自述：**离线、零依赖的 CLI。它针对一个目标文件计算 Codex、Claude Code、Cursor、GitHub Copilot、Gemini 的有效指令栈，列出来源文件、规则和冲突；可用 `--check` 或 `ci --strict` 作为 CI 门禁。
- **为什么值得关注：**编程 Agent 的失败有时来自互相矛盾的 `AGENTS.md`、`CLAUDE.md` 或路径规则，而非模型能力不足。将这类“隐形配置”转为可定位的诊断报告，有助于让自动化前提变得可审查。README 也明确指出当前检查偏保守，不能发现所有语义冲突。

## 趋势 / 编辑简评

今天的三个新仓库指向同一条工程主线：**先控制 Agent 的输入与循环，再让它的查询和规则可追溯。**从 `pi-for-each` 的会话隔离，到 Semantic Agent 的只读语义查询，再到 WhichRules 的指令冲突检查，关注点都不是“让 Agent 多做一点”，而是让团队知道它在什么边界内做了什么。它们都仍是很新的项目，适合用于概念验证和跟踪，不应直接视为生产就绪方案。
